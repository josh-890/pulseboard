# Image Media usage in Pulseboard

## General
Media in form of images is of central importance for the app. 

All images are anchored with **sessions**.
THere are two groups of sessions:
* production sessions:
 the record of collaboration and capture about the event in the real world during content creation.
* reference sessions:
 the record of all media refering and therefor uploaded to a specific person, with possibly unknown information about the creation

Sessions are the sources for media-items. Media-Items are used in various scenarios, 
as in:
* prodcution sets
a publication artifact that *references* to media in usually one **production session**. 
Production sets can be:
 	- Curated
 	- Re-released
 	- Re-labeled

* production compilations sets 
are a special form of production sets. They are Compiled from the creations of multiple **production sessions**.
A production compilatation set as publication artefact is connected to a **collaboration session**
on its own, hosting their own set of media-items. This session may reference to the creation productions sessions (if known). 

* profile sets
Profile sets are a selection of media-items to be shown in the picture gallery a persons profile.
Profile sets can be selected from the reference session of the respective person and from multiple production sessions the respective person has contributed to.

* headshot (sets)
media-Items selected from the reference session of a person and assigned to one of n slots. THe headshots are used as picture on the individual persons card in the peoples browser and in the hero section of a persons details page. 

* personal details sets
personal detail sets are a selection of media-items connected to a certain detail category of a specific person.
The are used to show up the specific "detail characteristic" gallery of a person.

The list of "detail categories" is open and changable. Categories can be added or removed and their labels can be changed. Therefore the tagging to the categories has to be generic and the respective label has to be set in a setting environment.
A details categorie can be, for example:

-- Physical characteristics
eyes, nose, belly button, ... 

-- Body Characteristics 
Scars, Marks, Tattoos, ...

-- Body Modifications
piercings and other jewelry mods, implants, brandings, ...

-- Body Procedures 
breast enlargement, botox lips, ...

Media connected to a detail categoy is shown on the persons details page in a suitable way. 

* user sets
user sets are a user defined selection of media-items to be shown in special theme oriented picture galleries (Favorites, Best of ..., special skills, Special themes). user sets can be selected from every media-item (reference or production)

* user article set
user defined selection to be used as image within an article (wikijs-like)
